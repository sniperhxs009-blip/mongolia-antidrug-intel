# 定稿运维说明（简易）

## 1. 关键配置（`.env`）

| 变量 | 建议值 | 说明 |
|------|--------|------|
| `SECRET_KEY` | ≥24 位随机串 | Render 已 `generateValue`；本地勿用默认值 |
| `NEWS_WHEN` / `FULL_WHEN` | `30d` | 检索时效，勿改回 `1y` 除非专项回溯 |
| `ENABLE_FORUM_SEARCH` | `false` | 论坛仅手动开启 |
| `CRAWL_LOOSE_FILTER` | `false` | 严格过滤 |
| `AUTH_ENABLED` | `true` | 生产必须开 |
| `AUTH_ADMIN_PASSWORD` / `AUTH_ANALYST_PASSWORD` | 强密码 | 管理员可采集/删除；研判员只读导出 |
| `EMAIL_ALERT_PORT` 等 | 可选 | 空则回退 `EMAIL_TO` |
| `REQUIRE_MANUAL_DEPLOY` | `true` | 禁止依赖自动部署钩子 |
| `CRAWL_FORBID_PROXY` | `true` | 禁止翻墙代理（硬性） |

## 2. 本地启动

```bash
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

浏览器打开 `/login` → 仪表盘。健康检查：`/api/system-health`（需登录）。

## 3. 数据库清理（脏数据）

- Web：管理员调用 `POST /api/intel/purge`（清理内蒙古/俄边境等无关项）
- 归档超 30 天非关键：

```bash
python scripts/archive_db.py
```

## 4. 备份与恢复

```bash
python scripts/backup_db.py backup
python scripts/backup_db.py restore data/backups/intel_backup_YYYYMMDD_HHMMSS.zip
```

备份含 SQLite、`reports/`、`audit/`；默认保留最近 14 份 zip。

## 5. 分级导出

登录后：

- `/api/export/ledger?fmt=csv`
- `/api/export/ledger?fmt=xlsx`
- `/api/export/ledger?fmt=docx&hide_details=true`（隐藏缴获数量与新型毒品细节）
- 报告页：`/reports/{id}?hide_details=1`

## 6. 分口岸趋势

- API：`GET /api/port-trend`
- 日报告正文已自动附带口岸趋势段落

## 7. 部署上线（人工）

1. 本地：`python -m pytest tests/test_core.py -q`
2. 确认无明文密钥提交
3. **人工** `git push` 触发 Render（勿配置自动 agent 部署）
4. Render 环境变量核对：`CRAWL_LOOSE_FILTER=false`、`NEWS_WHEN=30d`、`ENABLE_FORUM_SEARCH=false`、管理员密码已设置

## 8. 审计日志

路径：`data/audit/audit.jsonl`（或 `/var/data/audit/`）。默认留存 90 天，超期由 `purge_old_audit_logs` 清理。
