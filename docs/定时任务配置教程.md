# 定时任务配置教程

系统使用 APScheduler 常驻后台，应用启动即生效。

## 可选间隔

| 配置值 | 含义 |
|--------|------|
| `hourly` | 每小时全网巡检 |
| `every_6h` | 每 6 小时 |
| `every_12h` | 每 12 小时 |
| `daily` | 每 24 小时 |

## 配置方式

### 1. 环境变量（推荐生产）

在 `.env` 或 Render Environment 中设置：

```env
CRAWL_INTERVAL=every_12h
TIMEZONE=Asia/Ulaanbaatar
EMAIL_DAILY_BRIEF_HOUR=8
EMAIL_DAILY_BRIEF_MINUTE=0
```

### 2. Web 控制台

打开首页 →「任务控制」→ 选择间隔 →「更新定时」。

### 3. 内置任务

- `crawl_analyze`：按间隔执行「采集 + 日度研判 + 邮件」
- `daily_brief`：每天固定时刻再跑一轮（保证日简报）
- `weekly_report`：每周一 09:00（乌兰巴托时区）生成周报并推送

## 增量与断点

- 以 URL / 内容哈希去重，仅入库新增与内容变更
- 任务 `checkpoint` 记录已处理源，异常中断后可续跑
- 请求间隔 `CRAWL_REQUEST_DELAY_SEC`（默认 2.5–3 秒）降低封禁风险

## 手动触发

```bash
curl -X POST "http://127.0.0.1:8000/api/crawl/run?report_type=daily"
```
