# 定时任务配置教程

系统使用 APScheduler 常驻后台，应用启动即生效。定位为**长期新闻监测**：高频抓最新涉毒新闻，每日再做全量研判。

## 监测频率

| 配置值 | 含义 |
|--------|------|
| `every_30m` | 每 30 分钟新闻监测（更快） |
| `hourly` | 每小时新闻监测（推荐默认） |
| `every_6h` | 每 6 小时 |
| `every_12h` | 每 12 小时 |
| `daily` | 每 24 小时 |

## 两种任务模式

| 模式 | 做什么 | 何时跑 |
|------|--------|--------|
| `news` | 精简关键词 + `when:7d` 抓最新新闻，增量入库；新告警可邮件 | 按 `CRAWL_INTERVAL` 循环 |
| `full` | 全量搜索 + 交叉研判报告 + 日简报邮件 | 每天 `EMAIL_DAILY_BRIEF_HOUR:MINUTE` |

## 配置方式

### 1. 环境变量（推荐生产）

```env
CRAWL_INTERVAL=hourly
CRAWL_MODE=news
NEWS_WHEN=7d
TIMEZONE=Asia/Ulaanbaatar
EMAIL_DAILY_BRIEF_HOUR=8
EMAIL_DAILY_BRIEF_MINUTE=0
ENABLE_SEARCH_FEEDS=true
ENABLE_OFFICIAL_CRAWL=false
```

### 2. Web 控制台

打开首页 → 选择监测频率 →「更新监测频率」。

也可点：
- **立即抓取最新新闻**（`mode=news`）
- **全量巡检 + 日报告**（`mode=full`）

### 3. 内置任务 ID

- `news_monitor`：按间隔执行新闻监测
- `daily_full`：每天固定时刻全量研判 + 日简报
- `weekly_report`：每周一 09:00（乌兰巴托时区）周报

## 增量与去重

- 以 URL / 内容哈希去重，只入库新增与内容变更
- 指挥台每 30 秒轮询 `/api/intel?since_id=` 自动显示新条目
- 请求间隔 `CRAWL_REQUEST_DELAY_SEC` 降低封禁风险

## 手动触发

```bash
# 只抓最新新闻（快）
curl -X POST "http://127.0.0.1:8000/api/crawl/run?mode=news&report_type=daily"

# 全量巡检 + 报告
curl -X POST "http://127.0.0.1:8000/api/crawl/run?mode=full&report_type=daily"
```
